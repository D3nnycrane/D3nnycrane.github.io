Overview

Broken Access Control occurs when users can act outside of their intended permissions. Attackers can gain unauthorized access to sensitive data, functions, or administrative actions.

Example Exploit

Below is an example demonstrating a common access control issue:

// Example: URL manipulation allows unauthorized access
GET /admin/delete-user?id=123 HTTP/1.1
Host: vulnerable-app.local
Cookie: session=attacker-session

In this example, a user without admin privileges can delete other users by modifying the URL.

Proper Fix

  • Enforce server-side authorization checks for every action.
  • Implement a deny-by-default access policy.
  • Audit and log access control failures.
  • Use frameworks with built-in access control wherever possible.
